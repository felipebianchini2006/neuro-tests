## Architecture

Next.js 16 App Router (`src/` layout). Three URL spaces:

- `/admin` — password-protected admin dashboard (create sessions, watch status, delete completed)
- `/p/[token]` — participant view (takes the test)
- `/o/[token]` — observer/examiner view (watches participant progress in real time)

### Layer Stack

```
src/lib/domain/       Pure logic, zero dependencies
  cubes.ts            CubeFace type, rotation, tray building, board validation
  sequence.ts         SequenceStory type, shuffle, answer validation

src/lib/content/      Combines domain + static assets
  catalog.ts          All test content: 11 sequence stories + 9 cube challenges
                      Exports getTotalItems, getSequenceStoryAt, getCubeChallengeAt,
                      validateSequenceAnswer, validateCubeAnswer, getCubeTrayForSession
  sequence-manifest.generated.ts   Auto-generated by sync:sequence-assets — do not edit

src/lib/server/       Server-only (API routes + RSC)
  session-repository.ts   Dual-mode repository: in-memory Map OR Supabase postgres,
                          selected at runtime by getSessionRepository() based on env vars
  participant-session-state.ts  Builds ParticipantSessionState from a snapshot
  admin-auth.ts        Cookie-based admin session (ADMIN_PASSWORD env var)

src/lib/client/       Browser-only
  api.ts              fetch wrappers for session API routes
  session-channel.ts  Supabase Realtime broadcast per session token

src/components/
  admin/              AdminDashboard, AdminLoginForm
  participant/        SessionPlayer (orchestrator), SequenceSession, CubesSession
  observer/           SessionObserver (receives realtime snapshots)
  shared/             CubeFace (renders a single cube face SVG)

src/app/
  api/sessions/[token]/   start, answer, advance, complete
  api/admin/sessions/     list, create, complete, delete-completed
  api/admin/login|logout  cookie auth
```

### Session State Machine

`pending` → `in_progress` (first `start` call) → `completed`

Advancing only moves `currentItemIndex` forward when the current item `isCorrect === true`. The observer page subscribes to Supabase Realtime broadcast channel `session:${token}` to watch live progress.

### Dual Storage Mode

`getSessionRepository()` returns `supabaseRepository` when both `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` are set; otherwise falls back to `memoryRepository` (in-process `globalThis` Map, resets on server restart). Both implementations satisfy the same `SessionRepository` interface.

The browser Supabase client (for Realtime only) reads `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` or `NEXT_PUBLIC_SUPABASE_ANON_KEY`.

### Content Assets

Sequence stories are image files in `public/assets/sequence/<story-name>/`. The correct frame order is either explicit (hardcoded in `sync-sequence-assets.mjs`) or derived via a Python CV script (`scripts/derive-sequence-order.py`) that compares frames against a guide image. The generated manifest at `src/lib/content/sequence-manifest.generated.ts` is committed and must be regenerated when assets change.

Cube challenges are hardcoded in `catalog.ts` with their `CubeFace[][]` target grids. Reference images live at `public/assets/cubes/<n>.jpg`.

### Shuffle Determinism

Both tests use a seeded stable hash to shuffle per session token, ensuring the participant always sees the same initial scramble for a given session, and the observer can reconstruct it.

## Environment Variables

| Variable | Purpose |
|---|---|
| `ADMIN_PASSWORD` | Required to enable `/admin`. Without it, the panel shows a config error. |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL (optional) |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` or `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Realtime in browser |
| `SUPABASE_SERVICE_ROLE_KEY` | Server-side persistence; both URL + this key required to activate Supabase mode |

If Supabase is used, apply `supabase/schema.sql` to the project before running.
